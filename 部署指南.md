# 食帳 FOOD LEDGER — 部署指南

健身營養控制 PWA。部署後即擁有獨立網址，手機可「加入主畫面」變成 App 圖示，
相機／相簿權限正常運作，資料儲存於使用者手機本機（localStorage），離線亦可記錄。

---

## 一、檔案結構

```
foodledger/
├── index.html            # 主程式（單檔，含全部功能）
├── manifest.webmanifest  # PWA 設定（App 名稱、圖示、主題色）
├── sw.js                 # Service Worker（離線快取）
└── icons/
    ├── icon-192.png      # App 圖示（小）
    └── icon-512.png      # App 圖示（大）
```

---

## 二、部署方式（三選一，皆免費）

### 方式 A：Netlify Drop（最簡單，2 分鐘，免註冊帳號也可試用）

1. 開啟 https://app.netlify.com/drop
2. 把整個 `foodledger` 資料夾直接拖進網頁
3. 完成 — 立即獲得 `https://xxxx.netlify.app` 網址
4. （建議）註冊免費帳號可保留網站並自訂子網域名稱

### 方式 B：GitHub Pages（適合已有 GitHub 帳號者）

1. 建立新 repository（例如 `foodledger`）
2. 上傳資料夾內所有檔案到 repo 根目錄
3. Settings → Pages → Source 選 `main` branch → Save
4. 數分鐘後可用 `https://你的帳號.github.io/foodledger/` 開啟

### 方式 C：Vercel

1. https://vercel.com → New Project → 匯入資料夾或 GitHub repo
2. 不需任何設定，直接 Deploy

> ⚠ PWA 必須透過 **https** 網址使用（以上三種平台都自動提供）。
> 直接雙擊 index.html 用 file:// 開啟時，相機與離線功能不會生效。

---

## 三、安裝到手機主畫面

**iPhone（Safari）**
1. 用 Safari 開啟部署後的網址
2. 點底部「分享」按鈕 → 「加入主畫面」
3. 主畫面會出現黑底「食」字圖示，點開即全螢幕 App 體驗

**Android（Chrome）**
1. 用 Chrome 開啟網址
2. 右上角選單 → 「安裝應用程式」（或自動跳出安裝提示）

---

## 四、啟用 AI 拍照估算

拍照估算功能透過 Anthropic API 進行影像分析，需要一組 API Key：

1. 前往 https://console.anthropic.com → 註冊／登入
2. 左側 **API Keys** → **Create Key** → 複製金鑰（sk-ant- 開頭）
3. 開啟食帳 App → 「目標」分頁 → 底部「AI 拍照估算設定」→ 貼上金鑰

**費用說明**：依 Anthropic 官方按用量計費，分析一張餐點照片的成本極低
（單張約新台幣一角以內），一般個人使用每月費用通常只有數元台幣。

**隱私說明**：API Key 只儲存在使用者手機的瀏覽器本機空間，
不會上傳到任何第三方伺服器；照片僅於分析當下傳送至 Anthropic API。

---

## 五、資料儲存說明

- 所有紀錄（飲食、體重、目標、飲水）存於瀏覽器 localStorage
- 換手機或清除瀏覽器資料會遺失紀錄 — 屬單機版設計
- 若日後要做多裝置同步／會員系統，需加後端資料庫（可再擴充）

---

## 六、後續商業化擴充方向（白牌版本）

| 項目 | 修改位置 | 說明 |
|---|---|---|
| App 名稱 | manifest.webmanifest + index.html `<title>` | 改成教練工作室名稱 |
| 品牌色 | index.html `:root` CSS 變數 | 換主色即全站生效 |
| 圖示 | icons/ 兩張 PNG | 替換成客戶 Logo |
| 食物資料庫 | index.html 內 `FOOD_DB` 陣列 | 可加入教練指定餐單 |

一套原始碼可低成本複製給多位健身教練／營養師客製，
以「一次性建置費＋年度維護費」模式收費。
